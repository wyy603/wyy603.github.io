NOIP2018 游记

发表于 2018-11-11 | 分类于 OI |

$\text{Day 0}$

板子都在之前调过了。。所以一直在做题。

调了P1941 飞扬的小鸟调了差不多一天。。恶心的dp。。

$\text{Day 1}$

$8:00 - 12:30$ 提高Day1

$T1$原题啊。。现在CCF已经自己抄自己了吗。。半小时写完测完大样例就跑了

$T2$看起来很像大凯的疑惑。。又因为以前有几道数学题和这个差不多，再结合大样例猜了两个不会证的性质：

  • 答案的序列是给出序列的子序列

  • $x$能被表示当且仅当$x-a_1, x-a_2, .., x-a_n$至少一个能被表示。

于是一开始是针对$50$%的部分分：$O(T\times 2^n\times n\times \max{a_i})$，$2^n$是枚举给出序列的子序列，用$O(n\times max{a_i})$的复杂度将一个$vis[]$数组写出来，$vis[i]$表示$i$是否能被表示（这里因为感觉不稳就把上界调到了$n\times \max{a_i}$，反正部分分比较小就瞎搞。。），并与原序列能表示的数匹配，如果匹配成功$ans=min(ans, tans)$，$tans$是这个子序列的长度。

弄完之后感觉没啥可优化的。。就去看T3

啊T3是什么玩意。。感觉是树上dp，二分之类的，但碰巧我都不会。。于是就看部分分。

啊好像$M=1$，菊花图，链都比较可做。。于是就开了3个namespace，打完$M=1$的部分分后发现链的部分分似乎并不会做，又想了一个奇怪的贪心来做菊花图的部分分（这个贪心后来被发现是完全错误的），导致才写了两档部分分还有一档不稳。。

于是只好回来看T2。。准备对$n>5$的东西做玄学判定，又发现一个不会证的性质：

  • 当$a_i$可被表示时，它是一定会被删除的，而且不能被表示的$a_i$就一定不会删除

于是先用$O(n^3loga_i)$的复杂度筛掉了能被两个序列中的数表示的$a_i$，剩下的准备用刚才的算法。看起来总复杂度会变的比较玄学。

然后突然发现：判断一个$a_i$能否被表示，是能直接$n\times \max{a_i}$的复杂度用刚才的算法扫的。。

于是就写出了最后交上去的$O(Tn^2\times \max{a_i})$的代码。。

写完之后大概还有半小时，旁边的大佬直接提前离场了orz

剩下半小时就是各种测试。。T3也不会。。

最后大概就是这样了。。

预估分数：$100+80+20=200$

洛谷自测分数：$100+95+20=215$

大概遗憾就是T3部分分没拿好。。

$\text{Day 2}$

$8:00 - 12:30$ 提高Day2

爆炸的一天

先看$T1$，发现用邻接矩阵可能更好做一些，于是采用了邻接矩阵搜索，$O(nm)$过了$m=n-1$的部分，然后做了做感觉存在一条边使得砍掉这条边不影响答案，于是$O(nm^2)$写了$m=n$的部分

然后大概用了10几分钟想了想觉得并没有什么可以优化的地方，就去看$T2$

$T2$第一眼感觉是状压dp，真巧我状压dp不会。。于是就打$n=2$的部分分

暴力发现规律当$n=2$时$ans=4\times 3^{m-1}$，于是把这个式子打上去，顺便把样例全都放进了程序里期望骗分。。但我手残把$n,m$换了，考完后发现好像不能换。。。

看$T3$的时候看错题意，看成相邻节点只能有一个节点驻扎兵力。。还只打了链的暴力，后来才发现题意看错了，不过没法改了。。

打完$T3$的暴力错解后又回去把$T1$的$m=n$的部分优化成了$O(n(n+m))$。。然后又回来想$T3$其它部分分的解法因为我认为自己做不出$T2$的dp。。就这么丧失了$T2$推式子的机会

考完后听说$T2$答案是一个式子。。为什么我没有继续试下去啊orz
